## Authentication

All API endpoints require authentication using Bearer tokens. Include the token in the Authorization header:

```
Authorization: Bearer <your_jwt_token>
```

### Role-Based Access Control

The system implements strict RBAC with the following roles:

- **USER**: Can create reports and appeals
- **MODERATOR**: Can review reports, take actions, manage assigned cases
- **ADMIN**: Can manage cases, assign moderators, moderate trust scores
- **SUPER_ADMIN**: Full system access including user bans

## Base URL

```
https://api.knowledge-sharing-community.com/api/moderation
```

For development:
```
http://localhost:3000/api/moderation
```

## Rate Limiting

| Endpoint | Limit | Time Window |
|----------|-------|-------------|
| POST /reports | 5 requests | 1 hour |
| POST /appeals | 3 requests | 7 days |
| POST /cases/*/action | 20 requests | 1 hour |
| GET /dashboard | 100 requests | 1 hour |

## Error Handling

The API returns standard HTTP status codes with detailed error responses:

```json
{
  "success": false,
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Invalid input data",
    "details": {
      "field": "reason",
      "issue": "Minimum length is 10 characters"
    }
  }
}
```

### Common Error Codes

- `401` - Unauthorized (invalid or missing token)
- `403` - Forbidden (insufficient permissions)
- `404` - Not Found (resource doesn't exist)
- `409` - Conflict (duplicate report)
- `422` - Validation Error (invalid input)
- `429` - Too Many Requests (rate limited)
- `500` - Internal Server Error

## API Endpoints

### Reports

#### Create Report

Creates a new report against content or users.

```http
POST /reports
```

**Request Body:**
```json
{
  "reportedUserId": 123,
  "reportedPostId": 456,
  "reportedCommentId": 789,
  "severity": "MEDIUM",
  "flagType": "SPAM",
  "reason": "Content appears to be spam",
  "description": "The post contains repetitive and low-quality content",
  "evidenceUrls": [
    "https://example.com/evidence1.jpg",
    "https://example.com/evidence2.png"
  ]
}
```

**Required Fields:**
- `flagType` (enum): `SPAM`, `INAPPROPRIATE`, `HARASSMENT`, `MISINFORMATION`, `VIOLENCE`, `COPYRIGHT`, `OTHER`
- `reason` (string, 10-500 chars): Reason for the report
- At least one target: `reportedUserId`, `reportedPostId`, or `reportedCommentId`

**Optional Fields:**
- `severity` (enum): `LOW`, `MEDIUM`, `HIGH`, `CRITICAL` (default: `MEDIUM`)
- `description` (string, max 2000 chars): Detailed description
- `evidenceUrls` (array, max 5): Supporting evidence URLs

**Response:**
```json
{
  "success": true,
  "message": "Report submitted successfully",
  "data": {
    "report": {
      "id": 123,
      "severity": "MEDIUM",
      "status": "PENDING",
      "flagType": "SPAM",
      "reason": "Content appears to be spam",
      "isDuplicate": false,
      "autoGenerated": false,
      "createdAt": "2026-02-12T18:30:00Z",
      "updatedAt": "2026-02-12T18:30:00Z",
      "reportedUser": {
        "id": 123,
        "name": "John Doe",
        "email": "john@example.com"
      },
      "moderationCase": {
        "id": 456,
        "caseNumber": "CASE-2026-00001",
        "state": "OPEN"
      }
    }
  }
}
```

#### Get Report by ID

Retrieves a specific report.

```http
GET /reports/{id}
```

**Permissions:**
- Reporters can view their own reports
- Moderators+ can view any report

**Response:** Same structure as create response

#### Get All Reports

Retrieves a paginated list of reports.

```http
GET /reports
```

**Query Parameters:**
- `status` (enum): Filter by report status
- `severity` (enum): Filter by severity level
- `flagType` (enum): Filter by flag type
- `reportedUserId` (int): Filter by reported user
- `assignedModeratorId` (int): Filter by assigned moderator
- `page` (int, default: 1): Page number
- `limit` (int, default: 20, max: 100): Items per page

**Response:**
```json
{
  "success": true,
  "message": "Reports fetched successfully",
  "data": {
    "reports": [...],
    "meta": {
      "page": 1,
      "limit": 20,
      "totalReports": 150,
      "totalPages": 8
    }
  }
}
```

### Moderation Cases

#### Create Moderation Case

Creates a new moderation case (Admin+ only).

```http
POST /cases
```

**Request Body:**
```json
{
  "title": "Harassment Investigation",
  "description": "Multiple reports of user harassment",
  "priority": "HIGH",
  "assignedModeratorId": 123,
  "autoGenerated": false
}
```

**Response:**
```json
{
  "success": true,
  "message": "Moderation case created successfully",
  "data": {
    "moderationCase": {
      "id": 456,
      "caseNumber": "CASE-2026-00002",
      "title": "Harassment Investigation",
      "description": "Multiple reports of user harassment",
      "priority": "HIGH",
      "state": "OPEN",
      "autoGenerated": false,
      "createdAt": "2026-02-12T18:30:00Z",
      "reportCount": 0,
      "actionCount": 0
    }
  }
}
```

#### Assign Moderation Case

Assigns a case to a moderator (Admin+ only).

```http
POST /cases/{id}/assign
```

**Request Body:**
```json
{
  "assignedModeratorId": 123
}
```

#### Take Moderation Action

Takes an action on a moderation case.

```http
POST /cases/{id}/action
```

**Request Body:**
```json
{
  "actionType": "USER_SUSPENSION",
  "actionDetails": "Suspension for harassment",
  "targetUserId": 789,
  "durationDays": 7,
  "trustScoreChange": -20
}
```

**Action Types:**
- `WARNING`: Issue warning to user
- `CONTENT_REMOVAL`: Remove reported content
- `USER_SUSPENSION`: Suspend user (requires durationDays)
- `USER_BAN`: Permanently ban user (Super Admin only)
- `TRUST_DEDUCTION`: Adjust trust score
- `ESCALATE`: Escalate to admin review
- `NOTES_ONLY`: Add notes without action

#### Resolve Moderation Case

Resolves or closes a moderation case.

```http
POST /cases/{id}/resolve
```

**Request Body:**
```json
{
  "resolutionNotes": "Case resolved after investigation",
  "finalStatus": "RESOLVED"
}
```

### Appeals

#### Create Appeal

Creates an appeal against a moderation action.

```http
POST /appeals
```

**Request Body:**
```json
{
  "moderationCaseId": 456,
  "originalActionId": 789,
  "appealReason": "Unfair suspension",
  "appealDescription": "I believe the suspension was unjustified",
  "evidenceUrls": [
    "https://example.com/context.jpg"
  ]
}
```

#### Update Appeal Status

Updates appeal status and adds review notes (Moderator+ only).

```http
PATCH /appeals/{id}
```

**Request Body:**
```json
{
  "status": "APPROVED",
  "reviewNotes": "After review, the appeal is approved",
  "finalDecision": "reverse"
}
```

**Status Options:**
- `SUBMITTED`: Initial state
- `UNDER_REVIEW`: Being reviewed
- `APPROVED`: Appeal granted
- `DENIED`: Appeal rejected
- `ESCALATED`: Escalated for higher review

Retrieves trust score information (Admin+ only).

```http

```

**Response:**
```json
{
  "success": true,
  "message": "Trust score fetched successfully",
  "data": {
    "trustScore": {
      "id": 123,
      "userId": 456,
      "overallScore": 75,
      "trustLevel": "TRUSTED",
      "contentQualityScore": 80,
      "communityEngagementScore": 70,
      "reliabilityScore": 75,
      "lastCalculatedAt": "2026-02-12T18:30:00Z",
      "user": {
        "id": 456,
        "name": "John Doe",
        "email": "john@example.com"
      }
    }
  }
}
```

#### Update User Trust Score

Updates trust score components (Admin+ only).

```http
PATCH /trust-scores/{userId}
```

**Request Body:**
```json
{
  "contentQualityScore": 85,
  "communityEngagementScore": 80,
  "reliabilityScore": 90,
  "trustLevel": "TRUSTED"
}
```

**Trust Levels:**
- `VIP`: 90-100 points
- `TRUSTED`: 75-89 points
- `ESTABLISHED`: 60-74 points
- `NEW`: 40-59 points
- `RESTRICTED`: 20-39 points
- `SUSPENDED`: 0-19 points

## Webhooks

The system supports webhooks for real-time notifications:

### Configure Webhook

```http
POST /webhooks
```

**Request Body:**
```json
{
  "url": "https://your-app.com/webhooks/moderation",
  "events": [
    "report.created",
    "case.assigned",
    "action.taken",
    "appeal.submitted",
    "trust.updated"
  ],
  "secret": "your_webhook_secret"
}
```

### Webhook Events

#### report.created
```json
{
  "event": "report.created",
  "data": {
    "report": { /* Report object */ }
  },
  "timestamp": "2026-02-12T18:30:00Z"
}
```

```json
{
  "event": "case.assigned",

    "case": { /* Case object */ },
    "assignedModerator": { /* User object */ }
  },
  "timestamp": "2026-02-12T18:30:00Z"
```

## SDKs & Libraries

### JavaScript/Node.js

```bash
npm install @ksc/moderation-sdk
```

```javascript
import { ModerationClient } from '@ksc/moderation-sdk';

const client = new ModerationClient({
  apiKey: 'your_api_key',
  baseUrl: 'https://api.knowledge-sharing-community.com'
});

// Create a report
const report = await client.reports.create({
  reportedPostId: 123,
  flagType: 'SPAM',
  reason: 'Spam content detected'
});

// Get dashboard data
const dashboard = await client.dashboard.get();
```

### Python

```bash
pip install ksc-moderation
```

```python
from ksc_moderation import ModerationClient

client = ModerationClient(

# Get dashboard data
dashboard = client.dashboard.get()
```
